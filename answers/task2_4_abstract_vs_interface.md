# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|-----------|-------------|
| Все животные имеют имя и возраст, но звучат по-разному | Абстрактный класс | Есть общие поля (имя, возраст) — состояние, отношение «является» (кошка является животным). Абстрактный метод `sound()` для разного звучания |
| Некоторые объекты (документ, изображение, настройки) можно сериализовать | Интерфейс | Отношение «умеет» (умеет сериализоваться). Объекты не связаны иерархией, нужна множественная реализация |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | Абстрактный класс | Есть общие поля (скорость, топливо) — состояние, отношение «является» (автобус является транспортным средством). Абстрактный метод `move()` |
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi | Интерфейс | Отношение «умеет» (умеет подключаться к Wi-Fi). Объекты из разных иерархий, нет общего состояния |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | Абстрактный класс | Есть общее поле (цвет) — состояние, отношение «является» (круг является фигурой). Абстрактные методы `area()` и `perimeter()` |
| Класс должен одновременно уметь летать, плавать и ездить | Интерфейс | Нужна множественная реализация: `implements Flyable, Swimmable, Drivable`. Java не поддерживает множественное наследование классов |
